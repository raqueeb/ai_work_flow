# Gemma 4 E4B‑এ আপগ্রেড

Qwen 2.5‑1.5B ব্যবহার করার পর যদি আপনি আরও শক্তিশালী রিজনিং ও উচ্চতর সঠিকতা চান, তাহলে **Gemma 4 E4B** মডেলে পরিবর্তন করুন। এই গাইডে আপগ্রেডের ধাপগুলো বর্ণনা করা হয়েছে।

## কেন আপগ্রেড করবেন?

| কারণ | Qwen 2.5‑1.5B | Gemma 4 E4B |
|------|--------------|------------|
| প্যারামিটার | 1.5 B | ~4 B |
| রিজনিং গভীরতা | মৌলিক | চেইন‑অফ‑থট (CoT) |
| সঠিকতা (জটিল টিকিট) | ~85% | ~95% |
| মেমরি ব্যবহার | ~2 GB VRAM | ~6 GB VRAM |
| ইনফারেন্স গতি | দ্রুত | ধীর (গভীর রিজনিং) |

## পূর্বশর্ত

- **LM Studio** ইনস্টল করা এবং Qwen মডেল লোড করা।  
- **Python 3.11+** এবং প্রয়োজনীয় প্যাকেজ (`streamlit`, `requests`) ইনস্টল করা।

## ইনস্টলেশন ধাপ

1. **LM Studio**-এ **Discover** আইকন ক্লিক করুন।  
2. সার্চ বারে `Gemma-4B-Instruct` লিখে মডেলটি খুঁজে বের করুন।  
3. **GGUF** সংস্করণ (Q4_K_M) ডাউনলোড করুন।  
4. **Models** ট্যাবে গিয়ে **Load** করুন – সার্ভার আবার `http://localhost:1234/v1` এ চালু হবে।

## স্ক্রিপ্টে মডেল পরিবর্তন

সব ডেমো স্ক্রিপ্টে `MODEL_NAME` ভেরিয়েবল আছে। Qwen থেকে Gemma‑তে পরিবর্তন করতে:

```python
# app-baseline-class.py অথবা gemma-4-e4b-app-baseline-class.py
MODEL_NAME = "gemma-4b-instruct"   # অথবা "qwen2.5-coder-1.5b-instruct"
```

ফাইলটি সেভ করে স্ক্রিপ্ট পুনরায় চালান; অন্য কোনো কোড পরিবর্তন দরকার নেই।

## Gemma‑ভিত্তিক ডেমো চালানো

```bash
# Gemma‑এর দ্রুত ডেমো
python gemma-4-e4b-app-reasoning2.py

# Streamlit UI
streamlit run gemma-4-e4b-app-baseline-class.py
```
