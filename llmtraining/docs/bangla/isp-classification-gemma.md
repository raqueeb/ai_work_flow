# ISP ক্লাসিফিকেশন (Gemma 4 E4B)#

এই গাইডটি **Gemma 4 E4B** মডেল ব্যবহার করে ISP সাপোর্ট টিকিটকে ৫০টি পূর্বনির্ধারিত ISP কোডে (যেমন `ISP‑001`) ম্যাপ করে। Gemma‑এর গভীর রিজনিং জটিল বা অস্পষ্ট টিকিটে উচ্চতর সঠিকতা প্রদান করে।

## কীভাবে কাজ করে#

1. **কীওয়ার্ড রুল** – প্রথমে দ্রুত ডিটারমিনিস্টিক ম্যাচিং করা হয়।  
2. **Gemma ফ্যালব্যাক** – কীওয়ার্ড না মিলে, টিকিটটি Gemma‑এ পাঠিয়ে সর্বোত্তম `ISP‑XXX` কোড জেনারেট করা হয়।

## ডেমো চালানো#

```bash
# Streamlit UI (Gemma সংস্করণ)
streamlit run gemma-4-e4b-app-baseline-class.py
```

অথবা পিউর পাইথন সংস্করণ চালান:

```bash
python gemma-4-e4b-app-reasoning2.py   # চেইন‑অফ‑থট (CoT) সংস্করণ
```

## প্রত্যাশিত ইনপুট#

**Ticket** ফিল্ডে কাঁচা টিকিট টেক্সট পেস্ট করুন। ক্লাসিফায়ার নিম্নলিখিত তথ্য আউটপুট করবে:

- **প্রেডিক্টেড ISP কোড** (যেমন `ISP‑023`)।  
- **কনফিডেন্স স্কোর** (০‑ধ০০)।  
- **জাস্টিফিকেশন** – সিদ্ধান্তের সংক্ষিপ্ত ব্যাখ্যা।  
- **ফিল্ড ডিসপ্যাচ ফ্ল্যাগ** – জরুরি (L2+) টিকিটের জন্য লাল রঙে হাইলাইটেড।

## কাস্টমাইজ করা#

- **নতুন কীওয়ার্ড যোগ** – `gemma-4-e4b-app-baseline-class.py`‑এর `FIELD_ISP_CODES` ডিকশনারি আপডেট করুন।  
- **কাস্টম প্রম্পট** – `gemma-4-e4b-model_use_class.py`‑এ সিস্টেম প্রম্পট পরিবর্তন করে অতিরিক্ত ট্রাবলশুটিং স্টেপ বা রিকমেন্ডেশন জিজ্ঞাসা করুন।  
- **মডেল সুইচ** – `MODEL_NAME = "qwen2.5-coder-1.5b-instruct"` করে Qwen‑এ ফিরে যান, অথবা `MODEL_NAME = "gemma-4b-instruct"` রেখে Gemma ব্যবহার চালিয়ে যান।

## কখন এই লিগেসি গাইড ব্যবহার করবেন?#

- বিদ্যমান ডিপ্লয়মেন্ট রক্ষণাবেক্ষণ করছেন যা এখনও পুরনো ফাইল নাম ব্যবহার করে।  
- দ্রুত রেফারেন্স দরকার, নতুন লার্নিং‑পাথ গাইড না ব্যবহার করে।  

আরো আধুনিক, ধাপে‑ধাপে অনবোর্ডিং অভিজ্ঞতার জন্য **Qwen 2.5 দিয়ে শুরু করা** এবং **Gemma 4 E4B‑এ আপগ্রেড** গাইডগুলো দেখুন।

*একই মূল লজিক বজায় রেখে, গোপনীয়তা ও লোকাল ইনফারেন্সের সুবিধা উপভোগ করুন।*
