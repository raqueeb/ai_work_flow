# ISP ক্লাসিফিকেশন (Qwen 2.5)

এই গাইডটি **Qwen 2.5‑1.5B** মডেল ব্যবহার করে ISP সাপোর্ট টিকিটকে ৫০টি পূর্বনির্ধারিত ISP কোডে (যেমন `ISP‑001`) ম্যাপ করার পদ্ধতি ব্যাখ্যা করে। হাইব্রিড পদ্ধতি ব্যবহার করা হয়: প্রথমে কীওয়ার্ড রুল, তারপর LLM ফ্যালব্যাক।

## কীভাবে কাজ করে

1. **প্রি‑প্রসেসিং** – টিকিট টেক্সটকে লোয়ারকেসে রূপান্তর এবং বিশেষ চিহ্ন সরিয়ে ফেলা।  
2. **কীওয়ার্ড ম্যাচিং** – উচ্চ অগ্রাধিকার কীওয়ার্ডের তালিকা স্ক্যান করে সরাসরি কোড রিটার্ন করে (৯৯% কনফিডেন্স)।  
3. **LLM ক্লাসিফিকেশন** – কোনো কীওয়ার্ড না মিলে, টিকিটটি Qwen‑এ পাঠিয়ে সর্বোত্তম ISP‑XXX কোড জেনারেট করা হয়।  
4. **ফলাফল প্রদর্শন** – Streamlit UI তে কোড, সংক্ষিপ্ত ব্যাখ্যা এবং কনফিডেন্স স্কোর দেখানো হয়। জরুরি (L2+) টিকিটের জন্য **Dispatch Field Team** বাটন সক্রিয় হয়।

## ডেমো চালানো

```bash
# ডিপেন্ডেন্সি ইনস্টল (যদি না করা থাকে)
pip install streamlit requests

# Streamlit ড্যাশবোর্ড চালু
streamlit run app-baseline-class.py
```

ব্রাউজার `http://localhost:8501` এ খুলবে; টিকিট পেস্ট করে **Analyze Ticket** ক্লিক করুন।

### উদাহরণ

**টিকিট:** “আমার ONT‑এ রেড লাইট জ্বলছে, ইন্টারনেট ডাউন।”  

**ফলাফল:**  
- **ISP কোড:** `ISP‑001` – রেড লাইট / ফিজিক্যাল ফাইবার কাট  
- **কনফিডেন্স:** 99% (কীওয়ার্ড ম্যাচ)  
- **অ্যাকশন:** ফিল্ড ডিসপ্যাচ প্রয়োজন (L2)

## কাস্টমাইজেশন

- **নতুন কীওয়ার্ড যোগ** – `app-baseline-class.py`‑এর `FIELD_ISP_CODES` ডিকশনারি আপডেট করুন।  
- **প্রম্পট পরিবর্তন** – `model_use_class.py`‑এ সিস্টেম প্রম্পট পরিবর্তন করে অতিরিক্ত ট্রাবলশুটিং স্টেপ জিজ্ঞাসা করুন।  
- **Gemma‑এ সুইচ** – `MODEL_NAME = "gemma-4b-instruct"` করে স্ক্রিপ্ট রিস্টার্ট করুন।
