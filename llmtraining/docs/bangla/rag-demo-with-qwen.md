# Qwen 2.5 1.5B দিয়ে RAG ডেমো

এই ডেমোটি দেখায় যে কীভাবে LM Studio-এর মাধ্যমে Qwen 2.5 1.5B প্যারামিটার মডেল ব্যবহার করে একটি স্থানীয় (local) Retrieval-Augmented Generation (RAG) সিস্টেম তৈরি করা যায়। এই ডেমোটি একটি ছোট ISP-সম্পর্কিত টেক্সট কর্পাস ব্যবহার করে দেখায় যে কীভাবে প্রাইভেট ও অন-প্রেমিসে AI-পাওয়ার্ড সমাধান তৈরি করা যায় বাইরের ক্লাউড API-এর ওপর নির্ভরশীল না হয়ে।

## ওভারভিউ

Retrieval-Augmented Generation (RAG) দুটি ক্ষমতা একত্রিত করে:
- ভাষা মডেলের সেমান্টিক বোঝাপড়া
- বাইরের নথি/ডেটা থেকে তথ্য উদ্ধার

এই পদ্ধতির সুবিধা:
- মিথ্যা তথ্য (hallucination) কমে — উত্তর সরাসরি উদ্ধৃত নথিতে ভিত্তি করে
- পুনরায় প্রশিক্ষণের প্রয়োজন নেই — শুধু নথি আপডেট করলেই যথেষ্ট
- সমস্ত তথ্য স্থানীয়ভাবে রাখা যায় — কোনো ডেটা বাইরে যায় না
- Qwen 2.5 1.5B-এর মতো ছোট মডেলেও দক্ষতার সাথে কাজ করা যায়

## প্রয়োজনীয় সামগ্রী

1. **LM Studio** — Qwen 2.5 1.5B Instruct মডেল লোড করা থাকবে এবং `http://localhost:1234`-এ API সার্ভার চালু থাকবে
2. **Python 3.11+**
3. **প্রয়োজনীয় প্যাকেজ**:
   ```bash
   pip install sentence-transformers faiss-cpu numpy requests
   ```

## কীভাবে কাজ করে

1. **নথি লোড**: একটি ISP-সম্পর্কিত নমুনা টেক্সট ফাইল লোড করা হয়
2. **টেক্সট বিভাগ**: ভালো প্রসঙ্গ ধরে রাখার জন্য নথিটি ওভারল্যাপিং ছোট ছোট খণ্ডে ভাগ করা হয়
3. **এমবেডিং তৈরি**: `all-MiniLM-L6-v2` নামের একটি সেন্টেন্স ট্রান্সফরমার মডেল ব্যবহার করে প্রতিটি খণ্ডের ভেক্টর তৈরি করা হয়
4. **ভেক্টর স্টোর**: FAISS ব্যবহার করে ভেক্টরগুলো সংরক্ষণ করা হয় যাতে দ্রুত সদৃশতা অনুসন্ধান করা যায়
5. **ক্যোয়ারি প্রসেসিং**:
   - ব্যবহারকারীর প্রশ্নটিকে ভেক্টরে রূপান্তর করা হয়
   - ভেক্টর স্টোর থেকে সবচেয়ে মিলে যাওয়া ৩টি খণ্ড উদ্ধার করা হয়
   - উদ্ধৃত নথি ও প্রশ্ন দিয়ে একটি প্রম্পট তৈরি করা হয়
   - LM Studio-এর মাধ্যমে Qwen 2.5 1.5B-কে প্রম্পটটি পাঠানো হয়
   - জেনারেটেড উত্তরটি প্রদর্শিত করা হয়

## এন্টারপ্রাইজের উপকারিতা

- **গোপনীয়তা**: সমস্ত প্রসেসিং স্থানীয়ভাবে হয় — কোনো ডেটা নেটওয়ার্ক ছেড়ে যায় না
- **মূল্য-কার্যকর**: কোনো টোকেন-ভিত্তিক বিলিং নেই; বিদ্যমান হার্ডওয়্যারে চলে
- **স্বনির্ভরশীল**: নিজস্ব নথি সহ সহজেই আপডেট করা যায়
- **স্কেলেবল**: বড় জ্ঞানবাণীভাণ্ডারের জন্য খণ্ডের আকার ও উদ্ধার প্যারামিটার সামঞ্জস্য করা যায়
- **অনুগত**: তথ্য সার্বভৌমত্ব রক্ষার জন্য উপযুক্ত — নিয়ন্ত্রিত শিল্পগুলোর জন্য আদর্শ

## ডেমোতে অন্তর্ভুক্ত ফাইল

- `rag_demo_qwen.py`: মূল ডেমো স্ক্রিপ্ট
- `sample_isp_data.txt`: নমুনা ISP জ্ঞানবাণী (যদি না থাকে স্বয়ংক্রিয়ভাবে তৈরি হয়)
- `requirements.txt`: প্রয়োজনীয় প্যাকেজের তালিকা

## চালানোর নিয়ম

1. নিশ্চিত করুন LM Studio চলছে এবং Qwen 2.5 1.5B Instruct লোড করা আছে ও API সার্ভার `1234` পোর্টে সক্রিয়
2. প্রয়োজনীয় প্যাকেজ ইনস্টল করুন: `pip install -r requirements.txt`
3. স্ক্রিপ্ট চালান: `python rag_demo_qwen.py`
4. নমুনা ISP তথ্য নিয়ে প্রশ্ন করুন (প্রস্থানের জন্য `quit` লিখুন)

## নমুনা আউটপুট

```
Initializing RAG system with Qwen 2.5 1.5B...
Loading sample ISP data...
Creating embeddings and building vector index...
RAG system ready! Ask questions about ISP operations (type 'quit' to exit)

> ফাইবার কেটে যাওয়ার সাধারণ কারণগুলো কী?
Based on the retrieved context:
ফাইবার কেটে যাওয়ার সাধারণ কারণগুলো হলো:
1. নির্মাণ কাজ — খনন, ড্রিলিং এবং রাস্তার কাজ মাঝে প্রায়শঃ আন্ডারগ্রাউন্ড কেবল ক্ষতিগ্রস্ত হয়
2. প্রাকৃতিক দুর্যোগ — বন্যা, ভূমিকম্প ও ভূমিধ্বস ফাইবার লিঙ্কগুলোকে কেটে ফেলতে পারে
3. কিছুমতি জন্তু — ভোরে ওঠা এবং অন্যান্য জন্তু প্রায়শঃ কেবলের আবরণ চিবিয়ে ফেলে
4. ভ্যান্ডালিজম — ইচ্ছাকৃত কেবল কেটে ফেলা বা তামার অংশ চুরি করা
5. দুর্ঘটনাজনক ক্ষতি — রক্ষণাবেক্ষণ কাজ বা কৃষি কাজের সময় কেবল ক্ষতিগ্রস্ত হতে পারে

The local LLM processed this query using retrieved context from the ISP knowledge base.
```

## আপনার এন্টারপ্রাইজের জন্য কাস্টমাইজ করা

1. **নমুনা তথ্য পরিবর্তন**:
   - `sample_isp_data.txt`-কে আপনার নিজস্ব নথি (প্রযুক্তিগত ম্যানুয়াল, এসওপি, FAQ ইত্যাদি) দিয়ে প্রতিস্থাপন করুন
   - PDF, DOCX ইত্যাদির জন্য স্ক্রিপ্টটি বাড়ানো যেতে পারে

2. **বিভাগ প্যারামিটার সামঞ্জস্য**:
   - `chunk_size` ও `chunk_overlap`-কে আপনার নথির গঠন অনুযায়ী পরিবর্তন করুন
   - সংক্ষিপ্ত খণ্ড নির্ভুলতা বাড়ায়, বড় খণ্ডে বেশি প্রসঙ্গ দেয়

3. **উদ্ধার সুর সামঞ্জস্য**:
   - `top_k` পরিবর্তন করে বেশি বা কম খণ্ড উদ্ধার করুন
   - FAISS-এ ভিন্ন সদৃশতা মেট্রিক পরীক্ষা করুন

4. **প্রম্পট উন্নত করা**:
   - `generate_response()`-এর প্রম্পট টেমপ্লেটে নির্দেশনা, ভূমিকা বা বিন্যাস নির্দেশ যোগ করুন
   - আপনার এন্টারপ্রাইজ-নির্দিষ্ট নিয়ম LLM-এর আচরণের জন্য যুক্ত করুন

## পারফরম্যান্স বিবেচনা

- **এমবেডিং মডেল**: `all-MiniLM-L6-v2` সিপিইউ-ভিত্তিক এমবেডিং জেনারেশনের জন্য হালকা ও দ্রুত
- **ভেক্টর স্টোর**: FAISS হাজার হাজার নথির জন্যও দ্রুত সদৃশতা অনুসন্ধান করে
- **LLM অনুমান**: Qwen 2.5 1.5B LM Studio-এর মাধ্যমে ক্রেতা GPU বা আধুনিক CPU-তে দক্ষতার সাথে চলে
- **ব্যাচ প্রসেসিং**: একাধিক ক্যোয়ারির জন্য এমবেডিং জেনারেশন ব্যাচ করে থ্রুপুট বাড়ানো যেতে পারে

   - স্বয়ংক্রিয় নথি প্রসেসিং পাইপলাইন বাস্তবায়িত করুন

2. **ইন্টিগ্রেশন পয়েন্ট**:
   - CRM/ERP সিস্টেমের সাথে সংযোগ করে প্রেক্ষাপট-সচেতন উত্তর
   - হেল্পডেস্ক প্ল্যাটফর্মের সাথে একীভূত করে স্বয়ংক্রিয় টিকিট প্রতিক্রিয়া
   - অভ্যন্তরীণ অ্যাপ্লিকেশনের জন্য API এন্ডপয়েন্ট তৈরি করুন

3. **নিরীক্ষণ ও প্রতিক্রিয়া**:
   - প্রশ্ন ও উত্তর লগ করে মান নির্ধারণের জন্য
   - মানব-ইন-দ্য-লুপ প্রতিক্রিয়ার জন্য সিস্টেম বাস্তবায়িত করুন
   - উদ্ধার প্রাসঙ্গিকতা ও LLM নির্ভুলতার মেট্রিক ট্র্যাক করুন

4. **উন্নত বৈশিষ্ট্য**:
   - বিভিন্ন নথি প্রকারের জন্য ক্যোয়ারি রাউটিং যুক্ত করুন
   - প্রসঙ্গ-সচেতন ফলো-আপের জন্য কথোপকথন ইতিহাস বাস্তবায়িত করুন
   - প্রয়োজন অনুযায়ী বহুভাষিক সমর্থন যোগ করুন
