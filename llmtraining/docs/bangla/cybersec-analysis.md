# সাইবারসিকিউরিটি অ্যানালিসিস

লোকাল LLM ব্যবহার করে দ্রুত সন্দেহজনক নেটওয়ার্ক অ্যাক্টিভিটি বিশ্লেষণ, সম্ভাব্য হুমকি চিহ্নিতকরণ এবং প্রতিকারের পদক্ষেপ সাজেস্ট করা যায়—সবই ক্লাউডে ডেটা না পাঠিয়ে।

## কেন LLM ব্যবহার করবেন?

- **গোপনীয়তা** – সব প্যাকেট ক্যাপচার, লগ এবং এলার্ট আপনার মেশিনে থাকে।  
- **গতি** – রিমোট API‑এর জন্য অপেক্ষা না করে লগ স্নিপেটের তাৎক্ষণিক বিশ্লেষণ।  
- **প্রসঙ্গ‑সচেতন যুক্তি** – মডেল একাধিক লগ এন্ট্রি একত্রিত করে প্যাটার্ন খুঁজে বের করতে পারে যা নিয়ম‑ভিত্তিক IDS‑এর পক্ষে কঠিন।

## সাধারণ ওয়ার্কফ্লো

1. **লগ সংগ্রহ** – ফায়ারওয়াল লগ, IDS এলার্ট বা সিসলগ এন্ট্রির একটি স্নিপেট এক্সপোর্ট করুন।  
2. **মডেলকে প্রম্পট করুন** – ছোট প্রম্পট ব্যবহার করুন:

   ```
   আপনি একজন সাইবারসিকিউরিটি অ্যানালিস্ট। নিচের লগটি বিশ্লেষণ করুন এবং বলুন এটি কোনো সম্ভাব্য অনুপ্রবেশ নির্দেশ করে কি না, এবং একটি প্রতিকারের পদক্ষেপ সাজেস্ট করুন।
   {log_snippet}
   ```

3. **আউটপুট পর্যালোচনা** – LLM একটি সংক্ষিপ্ত রায় (যেমন, “সম্ভাব্য পোর্ট স্ক্যান সনাক্ত”) এবং একটি সুপারিশকৃত পদক্ষেপ (যেমন, “সোর্স IP 192.0.2.45 ব্লক করুন”) প্রদান করে।

## ডেমো স্ক্রিপ্ট

`gemma-4-e4b-cybersec_analysis.py` একটি প্রস্তুত‑চালানো উদাহরণ যা:

- একটি লগ ফাইল পড়ে।  
- বিষয়বস্তু LM Studio‑এর মাধ্যমে Gemma মডেলে পাঠায়।  
- একটি সংক্ষিপ্ত সিকিউরিটি মূল্যায়ন প্রিন্ট করে।

এটি চালান:

```bash
python gemma-4-e4b-cybersec_analysis.py
```

## অ্যানালাইজার এক্সটেন্ড করা

- **কনটেক্সট যোগ করুন** – আরও সমৃদ্ধ উত্তরের জন্য প্রম্পটে সাম্প্রতিক এলার্ট বা অ্যাসেট ইনভেন্টরি অন্তর্ভুক্ত করুন।  
- **কাস্টম প্রম্পট** – মডেলকে ফায়ারওয়াল রুল, IDS সিগনেচার বা ইনসিডেন্ট রিপোর্ট স্নিপেট তৈরি করতে বলুন।  
- **SIEM‑এর সাথে ইন্টিগ্রেট করুন** – আপনার SIEM ডিমান্ড‑অন বিশ্লেষণের জন্য স্ক্রিপ্টটিকে একটি ছোট API‑তে মোড়ান।
